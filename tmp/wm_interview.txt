 <h4>10.11 WM/EWM 모듈 인터뷰 표준 질문 25개 — 창고관리</h4>
 <ol>
 <li><strong>창고 수와 위치, 면적은?</strong>
 <ul>
 <li><strong>의도</strong>: WM 도입 범위 결정.</li>
 <li><strong>프로젝트 영향</strong>: 창고 5개 이상 = SAP EWM(Extended WM) 권고. WM은 단순 운영용.</li>
 <li><strong>타 모듈 연계</strong>: MM(재고), SD(출하).</li>
 </ul></li>

 <li><strong>저장 위치(Storage Location·Bin) 체계는?</strong>
 <ul>
 <li><strong>의도</strong>: Bin 관리 자동화.</li>
 <li><strong>프로젝트 영향</strong>: Bin 단위 관리 = EWM 활성화.</li>
 <li><strong>타 모듈 연계</strong>: MM.</li>
 </ul></li>

 <li><strong>입고-피킹-출하 자동화 수준은?</strong>
 <ul>
 <li><strong>의도</strong>: 창고 운영 효율.</li>
 <li><strong>프로젝트 영향</strong>: 자동화 50% 이하 = WMS 도입 ROI 큼.</li>
 <li><strong>타 모듈 연계</strong>: SD, MM.</li>
 </ul></li>

 <li><strong>RF·PDA·Voice Picking 등 모바일 도구 활용은?</strong>
 <ul>
 <li><strong>의도</strong>: 모바일 통합.</li>
 <li><strong>프로젝트 영향</strong>: RF 미운영 = 입출고 정확도 90% 이하. RF 도입 권고.</li>
 <li><strong>타 모듈 연계</strong>: 외부 모바일 시스템.</li>
 </ul></li>

 <li><strong>FIFO·LIFO·SLED 출고 룰 적용 자재는?</strong>
 <ul>
 <li><strong>의도</strong>: Picking Strategy 결정.</li>
 <li><strong>프로젝트 영향</strong>: SLED 적용 = WM Picking Strategy 별도 설정.</li>
 <li><strong>타 모듈 연계</strong>: MM(Batch), QM(SLED).</li>
 </ul></li>

 <li><strong>실사 주기와 차이 처리 룰은?</strong>
 <ul>
 <li><strong>의도</strong>: 재고 정합성.</li>
 <li><strong>프로젝트 영향</strong>: 실사 차이 ±2% 이상 = 마스터 정제 필수.</li>
 <li><strong>타 모듈 연계</strong>: MM, FI.</li>
 </ul></li>

 <li><strong>외주 창고(3PL) 운영 비중과 시스템 연계는?</strong>
 <ul>
 <li><strong>의도</strong>: 3PL 인터페이스.</li>
 <li><strong>프로젝트 영향</strong>: 3PL 비중 30% 이상 = EDI 인터페이스 신설.</li>
 <li><strong>타 모듈 연계</strong>: 외부 3PL.</li>
 </ul></li>

 <li><strong>크로스도킹·재포장 운영 비중은?</strong>
 <ul>
 <li><strong>의도</strong>: 고급 WM 기능 필요성.</li>
 <li><strong>프로젝트 영향</strong>: 크로스도킹 30% 이상 = EWM 필수.</li>
 <li><strong>타 모듈 연계</strong>: SD.</li>
 </ul></li>

 <li><strong>위험물·온도 관리 자재 비중은?</strong>
 <ul>
 <li><strong>의도</strong>: 특수 자재 관리.</li>
 <li><strong>프로젝트 영향</strong>: 위험물 = EH&amp;S 모듈 도입 검토.</li>
 <li><strong>타 모듈 연계</strong>: EH&amp;S, QM.</li>
 </ul></li>

 <li><strong>운송업체와의 데이터 연계 방식은?</strong>
 <ul>
 <li><strong>의도</strong>: TM 통합.</li>
 <li><strong>프로젝트 영향</strong>: TM 도입 검토.</li>
 <li><strong>타 모듈 연계</strong>: SD, TM.</li>
 </ul></li>

 <li><strong>월 평균 입고·출고 트랜잭션 수는?</strong>
 <ul>
 <li><strong>의도</strong>: 시스템 부하·라이선스.</li>
 <li><strong>프로젝트 영향</strong>: 월 10,000건 이상 = HANA 인프라 강화.</li>
 <li><strong>타 모듈 연계</strong>: MM, SD.</li>
 </ul></li>

 <li><strong>창고 KPI 5개 — 입고시간·피킹정확도·재고회전·결품률·인당처리량?</strong>
 <ul>
 <li><strong>의도</strong>: 창고 성과 측정.</li>
 <li><strong>프로젝트 영향</strong>: KPI 자동화 = Fiori 대시보드.</li>
 <li><strong>타 모듈 연계</strong>: BI.</li>
 </ul></li>

 <li><strong>VMI·Consignment·Just-in-Time 운영 자재는?</strong>
 <ul>

 <li><strong>프로젝트 영향</strong>: VMI 운영 = 외부 시스템 연계.</li>
 <li><strong>타 모듈 연계</strong>: MM.</li>
 </ul></li>

 <li><strong>로트·시리얼 추적 자재 비중은?</strong>
 <ul>
 <li><strong>의도</strong>: 추적성 보장.</li>
 <li><strong>프로젝트 영향</strong>: 로트 추적 100% = SAP Batch 관리 필수.</li>
 <li><strong>타 모듈 연계</strong>: MM, QM, 61장.</li>
 </ul></li>
