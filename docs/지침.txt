docs에 문서를 남길때는 파일먕을 {타이틀}_******md 로 한다
각 타이틀별로 글의 컨셉, 창작의도가 다르므로 메모리는 분리하여 관리한다
파일명 끝에 _v1·_v2 등 버전 suffix가 붙은 파일은 확정·백업판이다.  읽지도 마라. 없다고 생각해라. 수정·삭제·이동·rename·내용 인용 금지. 사용자가 명시 지시할 때만 손댄다.
이 룰은 앞으로 만들어지는 타이틀에 적용된다 (현재 ERP컨설팅 + 향후 신규 타이틀). 기존 타이틀(가디언·고추밭징크스·쭈렉·가이드모드·모드체험·브로의반성문·김삽의사이드프로젝트)은 체계 도입 전 자산이라 적용 외다.
수정이력 파일은 저장하지 않는다 (수정이력/ 폴더 사용 중단). D:\mydocs\개발준수사항.txt의 수정이력 저장 룰은 이 프로젝트에서 적용 외다.

[공급망컨설팅]
* 라인구성
A : 컨설팅가이드. 사실 기반. 학습 데이터(왜곡 금지)
B : 업무적으로(컨설팅시) 이루어지는 에피소드. 업무적 대화. 사무실에서 일어나는 대화(자유 창작 가능)
C : 예약(미정)

D : 서정적 서사
E : 썸. 호감. 관심. 매력. 애틋
F : 질투, 의심. 갈등. 흥분. 오해
G : 베드. 격정. 19금
H : 거의베드. 헛물. 허탈.

1 : 핵절제
2 : 적절
3 : 핵상세
