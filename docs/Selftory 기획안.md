## 1. 서비스 컨셉

### 핵심 아이디어
> "내가 주인공이 되는 맞춤형 AI 소설 생성 서비스"
> *"모든 인간은 자기 인생의 미완성 소설이다. Selftory는 그 소설의 펜을 쥐어준다."*

사용자가 자신의 캐릭터(성격, 고민, 상황, 외모, 직업 등)를 입력하면, AI가 그 사람을 주인공으로 한 소설을 생성한다.

### 핵심 가치: 감정이입 × 창작이입 × 치유
- 단순히 "소설을 써주는 도구"가 아님
- **"내가 주인공"** — 감정이입: 읽으면서 "이거 나잖아" 하는 몰입감
- **"내가 작가"** — 창작이입: 소설 진행 중 독자에게 선택권을 부여하여, 마치 본인이 쓴 것 같은 환각을 만듦. 독자는 단순 독자가 아니라 공동 작가임
- 감정이입과 창작이입이 섞이면서 완전한 몰입이 발생함
- 클로드한테 "소설 써줘"는 남의 이야기지만, Selftory는 **내가 쓴 내 이야기**
- 자기 이야기를 풀어내는 과정 자체가 **회고, 자기반성, 정서적 치유**로 이어짐
- 엔터테인먼트를 넘어선 **심리적 위안과 치유의 도구**

### 치유의 메커니즘
> *"인간은 자신의 그림자를 직시할 때 비로소 빛을 이해한다."* — 카를 융의 그림자 이론에서 영감

- 인간은 본질적으로 객관화되어 있지 않다. 내면에는 추악함, 모순, 나약함이 있다
- 이를 억압하면 병이 되고, 소설이라는 안전한 공간에서 **대리 표출**하면 해소가 된다
- 표출 → 해소 → 위안. 이것이 Selftory가 제공하는 치유의 구조
- 소설은 인간의 모든 면을 다룬다. 성적인 것, 폭력적인 것, 조소, 비하 — 이 모든 것이 문학의 재료이며, 이를 허용해야 진정한 치유가 가능하다
- **표현의 자유는 치유의 전제조건이다** — 표현을 제한하면 치유의 가치가 훼손된다
- 수위 조절은 AI가 문학적 맥락 안에서 자체적으로 수행한다
- Selftory의 콘텐츠는 **문학과 철학의 영역**에 속한다. 문학은 인간 존재의 모든 측면을 탐구할 권리가 있으며, 철학은 그것에 의미를 부여한다. 이 두 가지가 Selftory의 표현 자유를 뒷받침하는 근거이다

> *"우리는 타인의 이야기에 울지만, 자신의 이야기 앞에서는 비로소 치유된다."*

### 일반 AI 대비 차별점
| 구분 | 클로드/ChatGPT 직접 사용 | Selftory |
|------|------------------------|----------|
| 프롬프트 | 잘 써야 함 | UI로 쉽게 입력 |
| 설정 | 매번 다시 설명 | 저장/관리 |
| 결과물 | 들쭉날쭉 | 최적화된 프롬프트 노하우 적용 |
| 히스토리 | 없음 | 내 소설 관리, 이어쓰기 |
| 진입장벽 | 높음 (구독 필요, 프롬프트 지식) | 낮음 (원클릭) |
| 커뮤니티 | 없음 | 다른 사람 소설 구경, 공유 |

## 2. 타겟 사용자

### 주인공 캐릭터 예시 (브레인스토밍 원본)
> 40대 중반 시니어 ERP 개발자. 20년차, 특출나진 않지만 가끔 필 받으면 두각을 보인다. 가정, 외모, 재산 다 그럭저럭. 빚 있고, 용기 없고, 나태하지만 매너는 지킴. 겉과 속이 다른 강약약강의 내면. 중년의 허탈감, 미래/건강/노후 걱정.

### 핵심 인사이트
- 사람들은 숨기고 싶은 강약약강의 내면이 있다
- 겉으로는 강한 척 하다가 속은 약하고, 약한 모습 보이다가 의외의 순간에 강함
- 많은 걸 극복하고, 학습에 의해 매너를 지키며 살아간다
- **이 컨셉이면 많은 사람들이 공감할 수 있다**

### 타겟층
- 30~50대 직장인 (중년의 허탈감, 미래 걱정에 공감하는 세대)
- 소설/웹소설 독자 (자기 이야기를 읽고 싶은 사람)
- AI 서비스에 익숙하지 않은 일반 사용자 (진입장벽을 낮춰주는 것이 가치)

## 3. 주요 기능

### MVP (최소 기능)
1. **캐릭터 설정 입력**
   - 성별, 나이, 직업, 성격, 외모
   - 현재 고민/상황
   - 강점/약점
   - 내면의 모순 (겉 vs 속)

2. **장르/분위기 선택**
   - 로맨스, 액션, 코미디, 드라마, 막장, 감동, 스릴러
   - 복합 장르 지원 (예: 기쁨 + 해학 + 감동 + 재미 + 반전 + 액션 + 멜로)

3. **소설 생성**
   - 단편 (1회 완결)
   - 장편 (챕터별 생성, 이어쓰기)

4. **공동 창작 (핵심 차별점)**
   - 소설 진행 중 주요 분기점마다 독자에게 선택권 부여
   - 단순 A/B 분기가 아닌, 선택이 이후 스토리 전체 방향에 영향
   - 독자의 선택이 누적되어 결말을 결정 → "내가 쓴 소설" 환각 유도
   - 한번에 결말까지 생성하는 옵션도 제공

5. **내 소설 관리**
   - 히스토리 저장
   - 이어쓰기
   - 분기 이력 확인 (내가 어떤 선택을 했는지)

### 창작 모드 (사용자 레벨별)

1. **자유 모드** — 빈 캔버스. 사용자가 힌트를 던지면 AI가 받아침. 창조력 높은 사용자용.
2. **가이드 모드** — AI가 먼저 질문. "주인공 이름은?" "어떤 상황?" "누가 배신하면 좋겠어요?" 선택지로 힌트를 유도. 일반 사용자용.
3. **분기 모드** — AI가 소설을 쓰다가 멈추고 A/B/C 선택지 제공. 고르기만 하면 됨. 소극적 사용자용.

어떤 레벨이든 "내가 만들었다"는 환각은 동일. 창작이입의 본질은 모드와 무관.

### 확장 기능 (추후)
- 커뮤니티 (다른 사람 소설 구경, 공유, 이어쓰기)
- 전자책 변환/다운로드
- 표지 이미지 AI 생성
- SNS 공유용 카드뉴스 변환
- 오디오북 변환 (TTS)

## 5. 수익 모델

### 핵심 전환: 완성품이 아니라 과정을 판다

> *Selftory는 소설 생성 서비스가 아니라, AI와 함께하는 인터랙티브 스토리 게임이다.*
> *노래방이 노래를 파는 게 아니라 부르는 경험을 파는 것처럼, Selftory는 만드는 과정을 판다.*

사용자는 소설을 "읽는" 것이 아니라 "플레이"한다. 힌트를 던지고, 방향을 틀고, 감정을 느끼고, 또 던진다. 이 과정 자체가 상품이다.

### 메인 수익: 세션 과금
- 1세션 (1~2시간 공동 창작): 3,000~5,000원
- 무료 체험: 월 1세션
- 세션 중 AI와 실시간으로 힌트를 주고받으며 소설을 함께 만듦
- 세션이 끝나면 완성된 소설이 자동 저장됨

### 부가 수익 1: 소장용 전자책
- 완성된 소설을 전자책(ePub/PDF)으로 변환: 1,000원
- 표지 이미지 AI 생성: 500원
- 오디오북 변환 (TTS): 2,000원

### 부가 수익 2: 프리미엄 모드
- 병맛 모드 / 막장 모드 / 문학 모드 / 로맨스 모드 — AI 반응 스타일 선택
- 프리미엄 모드: 월 9,900원 (AI가 더 깊이 반응, 캐릭터 심리묘사 강화, 분기 확장)

### 부가 수익 3: 커뮤니티 이어쓰기
- 내가 만든 소설을 공유 → 다른 사용자가 이어쓰기 가능
- 이어쓰기 세션당 3,000원, 원작자에게 수익 30% 분배
- 하나의 소설이 여러 사용자의 공동 창작물로 확장

### 부가 수익 4: SNS/공유
- SNS 공유용 카드뉴스 변환
- 명장면 이미지 생성 (AI)

### 기존 방안 (보조)

#### 방안 A: 광고 모델
- 무료 세션 + 세션 전후 광고 삽입
- 광고 제거는 프리미엄 구독에 포함

#### 방안 B: 건당 완성본 과금
- 세션 없이 AI가 자동 생성한 소설: 단편 500원, 장편 2,000원
- 공동 창작이 아닌 일방적 생성 (가치가 낮으므로 저가)

### 추천 조합
**세션 과금(메인) + 프리미엄 구독 + 전자책/커뮤니티(부가)**. 무료 1세션으로 맛보게 하고, 과정의 중독성으로 재구매를 유도하는 구조.

## 6. 기술 스택 (검토 사항)

### AI 모델
- **Haiku** (비용 효율): 월 100만건 기준 ~$5,000
- **Sonnet** (품질/비용 균형): 월 100만건 기준 ~$60,000
- **Opus** (최고 품질): 비용 감당 불가

### 현실적 접근
- 처음엔 Haiku로 시작
- 사용자 늘어나면 단계적으로 확장
- 소설 길이/품질에 따라 모델 선택 가능 (무료=Haiku, 유료=Sonnet)

### 처리량
- API 자체는 가능하지만 높은 티어 필요
- 동시 요청 수 제한 → 큐 시스템 필요

> 기존 서비스는 전부 "소설을 써주는 도구"이지, "내 캐릭터를 입력하면 내가 주인공이 되는 소설"은 명확한 포지션을 잡은 곳이 없다.
>
> *"기술은 도구를 만들고, 철학은 의미를 만든다. Selftory는 둘 다를 만든다."*

## 8. 콘텐츠 책임 정책

### 책임 구조
| 주체 | 역할 | 책임 범위 |
|------|------|-----------|
| 사용자 | 캐릭터/장르 등 입력 제공 | 입력 내용에 한정 |
| Selftory | 입력 전달, UI 제공, 중개 | 서비스 운영, 필터링 |
| Anthropic (Claude API) | 소설 실제 생성 | 생성 콘텐츠에 대한 최종 책임 |

### 핵심 원칙
- **표현의 자유 보장** — 사용자의 창작 표현을 검열하지 않음. 창작 과정에서의 자유로운 표현이 정서적 치유의 핵심이며, 이를 제한하는 것은 서비스의 본질적 가치를 훼손함
- **콘텐츠 책임은 AI 제공자(Anthropic)에게 있음** — AI가 생성한 콘텐츠의 최종 책임은 생성 주체에 있음. 사용자는 힌트를 입력했을 뿐이고, Selftory는 중개 플랫폼일 뿐임. 사용자에게도 Selftory에게도 책임을 전가하지 않음
- Selftory는 서비스 약관에 이 책임 구조를 명확히 명시할 것

## 11. 참고: 시범 생성 소설

브레인스토밍 과정에서 실제로 장편 소설을 생성하여 품질을 검증함.

- **제목:** 쭈렉의 코사무이 대모험
- **장르:** 코미디 + 로맨스 + 반전 + 감동
- **주인공:** 사용자가 제공한 힌트를 기반으로 자동 생성
- **결과:** 10장 + 에필로그, 모든 감정 요소 포함
- **위치:** 별도 파일로 보관
